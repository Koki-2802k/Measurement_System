# リアルタイムGPS測定機能の統合仕様書

## 1. 概要
本システムは，Movella DOTセンサーによるボート競技の動作計測（3台同期）と並行して，GPSモジュールを用いた位置情報（緯度，経度）および速度のリアルタイム測定を統合するものです．
取得されたGPSデータは，センサーのメタデータとともにデータ切り分けモジュールに渡され，ストロークごとのデータとしてCSVに分割出力されます．

## 2. 目的
- センサーからの加速度・角速度・クォータニオン情報に加え，位置・速度情報を統合してストローク単位での分析を行う．
- 将来のGPSモジュール導入に向け，非同期でデータを取得・統合できるスレッドセーフなアーキテクチャを構築する．

## 3. モジュール構成

### 3.1 `get_gpsdata.py` (GPSデータ取得モジュール)
- **クラス**: `GPSReader`
- **機能**:
  - GPSモジュールからデータを非同期（別スレッド）で取得する．
  - 現在は暫定としてダミーデータ（緯度，経度，速度）を1Hzで生成する．
  - 将来的にモジュールが決定次第，`_run`メソッド内の処理を実際の機器からの読み取り処理（例：シリアル通信など）へ書き換えるだけで対応可能とする．
- **出力形式**:
  - 辞書のリスト：`[{'time': datetime, 'latitude': float, 'longitude': float, 'speed': float}, ...]`
- **堅牢性機能**:
  - `_run` ループ内の例外ハンドリング（通信エラー時にスレッドがクラッシュしない）
  - GPS信号喪失時に直前の有効データをフォールバックとして使用する機構
  - `is_healthy` プロパティにより，メインスレッドからスレッドの正常動作を監視可能
  - 連続エラーが閾値（10回）を超えた場合に `is_healthy = False` となり警告を出す
  - `stop()` 時のタイムアウト付き `join`（5秒）によりスレッドの安全な停止を保証

### 3.2 `movelladot_pc_sdk_save_csv.py` (計測・保存スクリプト)
- **変更点**:
  - `GPSReader` をインポートし，センサーのロギング（Phase 2）と同時にスレッドを起動する．
  - チャンクサイズ到達時にバッファが切り出されるタイミングで，`GPSReader`から最新のGPSデータを取得する．
  - 取得したGPSデータを `Datadivision.py` の `DataProcessor` インスタンスへ受け渡す．
  - 終了時にGPS読み取りスレッドを適切に停止（`stop()`メソッド）する．
- **安定性改善**:
  - **パケットロス監視**: 受信サイクルごとにパケットの成功/失敗をカウントし，コンソールにリアルタイムでロス率を表示．終了時に統計サマリーを出力する．
  - **GPS健全性チェック**: チャンク処理ごとに `gps_reader.is_healthy` を確認し，異常時にコンソール警告を表示する．
  - **堅牢な終了処理**: `try/except/finally` 構造により，`KeyboardInterrupt` だけでなく予期しないエラーが発生した場合でも，GPS停止・CSVクローズ・センサ停止・同期停止が必ず実行される．

### 3.3 `Datadivision.py` (データ切り分けモジュール)
- **変更点**:
  - `DataProcessor` クラスに `add_gps_data` メソッドを追加し，動的に取得されたGPSデータリストを `locate_dict` へ登録する．
  - ストローク毎のCSV分割処理において，`get_locate(time)` を用いて対象時刻のGPS情報を照会し，ファイルに出力する．
- **安定性改善**:
  - `@lru_cache` を廃止し，手動キャッシュ（`_locate_cache` 辞書）に置き換え．`add_gps_data()` 呼び出し時にキャッシュを自動クリアすることで，動的に変わる `locate_dict` に対して古い結果が返されるバグを解消．

## 4. 今後の作業

- **データ周波数や精度調整**: 実機に合わせ，スリープ処理を含めたスレッドの読み取り間隔や丸め誤差（`10ms`など）の適正化を図る．
